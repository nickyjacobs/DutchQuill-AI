# Metadata-opschoning — Hoe het werkt

DutchQuill AI schoont automatisch documentmetadata op zodat gegenereerde .docx- en .pdf-bestanden niet herleidbaar zijn naar python-docx, ReportLab of automatische generatie. Dit document legt uit wat er wordt opgeschoond, waarom, en welke detectietools scholen gebruiken.

---

## Waarom metadata-opschoning

Plagiaatdetectietools als Turnitin analyseren niet alleen tekst, maar ook documentmetadata. Turnitin's Document Details toont het softwareprogramma, de auteur, bewerkingstijd en versie-informatie van ingeleverde bestanden. python-docx en ReportLab laten herkenbare fingerprints achter in deze metadata.

---

## Wat wordt opgeschoond

### .docx bestanden (word_export.py)

| Veld | python-docx default | Na opschoning | Reden |
|------|---------------------|---------------|-------|
| `AppVersion` | 14.0000 (Word 2011) | 16.0000 (Word 365/2021) | Versie 14 is onrealistisch voor een student in 2026 |
| `TotalTime` | 0 minuten | 15–90 minuten (variabel) | Een bewerkingstijd van 0 is verdacht |
| `Author` | "python-docx" | Studentnaam uit metadata | Toont wie het document heeft gemaakt |
| `Last Modified By` | Leeg | Studentnaam uit metadata | Moet overeenkomen met de auteur |
| `Description` | "generated by python-docx" | Leeg | Directe python-docx fingerprint |
| `Revision` | 1 | 8–25 (variabel) | Revisienummer 1 duidt op eerste generatie |
| `Created` | Generatietijdstip | Offset van 2–48 uur eerder | Identieke created/modified is verdacht |
| `Modified` | Generatietijdstip | Offset van 5–30 minuten eerder | Moet later zijn dan created |

De opschoning werkt in twee stappen:

1. **core.xml** — python-docx biedt directe API-toegang via `doc.core_properties`. Alle velden worden ingesteld voordat het bestand wordt opgeslagen.
2. **app.xml** — python-docx biedt geen API voor dit bestand. Na `doc.save()` wordt het .docx-bestand (een ZIP-archief) geopend en wordt app.xml herschreven met realistische waarden.

### .pdf bestanden (generate_report_pdf.py)

| Veld | ReportLab default | Na opschoning | Reden |
|------|-------------------|---------------|-------|
| `Author` | "DutchQuill AI" | Leeg | Mag geen toolnaam bevatten |
| `Creator` | "ReportLab" | Leeg | Verwijst naar de PDF-bibliotheek |
| `Producer` | "ReportLab PDF Library" | Leeg | Verwijst naar de PDF-bibliotheek |
| `Title` | "... — DutchQuill AI" | Alleen rapporttitel | Geen toolnaam in de titel |

De visible footer in het PDF-rapport ("Gegenereerd door DutchQuill AI") blijft ongewijzigd. Dit zijn interne analyserapporten die niet worden ingeleverd bij school.

## Wat NIET verdacht is

De volgende metadata hoeft niet te worden aangepast:

- **"Microsoft Macintosh Word"** als Application — standaard voor Word op macOS
- **"macOS Version X.Y Quartz PDFContext"** als Producer in PDF — standaard voor elke PDF die op een Mac wordt geëxporteerd
- **"Normal.dotm"** als Template — standaard Word-template
- **A4-paginagrootte** — standaard in Nederland
- **Times New Roman, Calibri, Cambria, Courier New** — standaard academische lettertypen
- **Claude/Anthropic in bijlage-tekst** — transparante APA-citatie over AI-gebruik, vereist door het beleid van Nederlandse hogeronderwijsinstellingen

## Detectietools die scholen gebruiken

### Turnitin (meest relevant)

Turnitin is de standaard plagiaatscanner bij vrijwel alle Nederlandse universiteiten en hogescholen. Het bevat twee relevante functies:

**Document Details** — Toont metadata van ingeleverde bestanden:

| Veld | Wat Turnitin toont | Wat verdacht is |
|------|-------------------|-----------------|
| Author Name | Naam van de bestandscreator | Naam die niet overeenkomt met de student |
| Last Modified By | Laatste bewerker (alleen .docx) | Andere naam dan de auteur |
| Document Software | Applicatie + versie | Ongebruikelijke software (Kingsoft, WPS Office) |
| Page Sizes | A4, US Letter, of anders | Ongebruikelijk formaat voor de regio |
| Fonts | Lettertypen, groottes, kleuren | Inconsistente opmaak (wijst op kopiëren) |
| Spelling | American vs. British English verdeling | Gemengde spelling wijst op meerdere bronnen |

**AI Writing Detection** — Analyseert tekstpatronen (perplexity, burstiness) om AI-gegenereerd schrijven te identificeren. Sinds eind 2025 detecteert het ook "gehumaniseerde" tekst die door bypass-tools is verwerkt.

### Andere tools

| Tool | Wat het analyseert | Metadata-check? |
|------|-------------------|-----------------|
| GPTZero | Tekstpatronen (perplexity/burstiness) | Nee |
| Scribbr AI Detector | Tekstpatronen | Nee |
| Copyleaks | Tekstpatronen + metadata via API | Ja (issuer, creation time, tool used) |
| Ouriginal/Urkund | Plagiaatdetectie (tekst) | Nee (wordt Turnitin per juni 2026) |
